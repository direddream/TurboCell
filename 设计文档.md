# TurboCell 极速芯研 Web 原型（浅色企业风）——MD 需求说明（给 Claude/设计/前端）

> 目标：做一套“可路演演示”的平台外观（不强求真实算法），让用户看到 **输入 → 分析 → 策略 → 报告交付** 的完整闭环。  
> 三条产品线命名：**Turbo-Select / Turbo-Dev / Turbo-Inspect**（统一产品化叙事）。

---

## 1. 产品定位与演示目标

### 1.1 一句话定位
TurboCell 是面向电池全生命周期的工程智能平台，提供选型与边界评估、研发加速、在役/退役评估与诊断的标准化交付。

### 1.2 路演演示要做到的“3个瞬间”
1) 改一个参数，结果立刻变化（哪怕是假数据联动）。  
2) 右侧结论卡给出“可执行策略”（参数表/边界/分流建议）。  
3) 一键导出报告（PDF/数据表占位下载）。

---

## 2. 信息架构（IA）

### 2.1 顶部导航（Top Bar）
- 左：Logo + 当前项目下拉（Project Switcher）
- 中：全局搜索（项目/电芯/报告）
- 右：演示模式开关（Demo Mode）｜导出中心（Exports）｜账号/权限（Account）

### 2.2 首页（Home）
- 三条产品线入口卡：Turbo-Select / Turbo-Dev / Turbo-Inspect
- 一分钟案例（Demo Cases）3个按钮
- 最近交付（Recent Deliverables）报告卡片 3–6 个

### 2.3 进入任一产品线后的侧边栏（Sidebar）
固定四段流程（所有产品线保持一致，降低学习成本）：
1) 输入 Input  
2) 分析 Analyze  
3) 策略 Strategy  
4) 报告 Report

---

## 3. 视觉风格（浅色企业风）

### 3.1 设计原则
- 清爽、克制、可信：留白多、卡片清晰、信息层级强
- “结论优先”：右侧结论卡始终可见
- 组件统一：相同交互在不同产品线一致

### 3.2 排版与网格
- 页面最大宽度：1200–1440（桌面优先），左右留白
- 12列栅格；卡片圆角小（6–10px），阴影极弱
- 字体建议：中文用系统字体（苹方/思源黑体/微软雅黑），英文用 Inter/Roboto

### 3.3 颜色与状态（建议，可由设计替换）
- 主色（Primary）：企业蓝系（用于按钮、链接、关键高亮）
- 辅助色：信息灰（文本/边框），成功绿、警告橙、风险红（仅用于标签/状态）
- 背景：浅灰白（减少纯白刺眼）
- 风险表达：避免大面积红色，用“标签 + 小图标 + 轻背景”表达

---

## 4. 通用页面布局模板（强建议统一）

### 4.1 三栏布局（最适合路演）
- 左栏：输入与参数（可折叠，宽 280–320）
- 中栏：主结果可视化（榜单/曲线/热力图/证据链）
- 右栏：结论卡 + CTA（一键导出 / 下一步）（宽 320–360）

### 4.2 通用组件库（必须做）
- Project Card（项目卡）
- KPI Tiles（关键指标块：寿命区间/风险等级/适配度）
- Risk Tag（风险标签：低/中/高）
- Evidence Chain（证据链组件：现象→指标→推断）
- Stepper（流程步骤条：提交→计算→审核→交付）
- Report Builder（报告生成器：模板+章节勾选+导出）
- Empty / No Match 状态（引导到“定制服务/升级VIP”）

---

## 5. 页面清单（9页路演原型）

> 每条产品线 3 页，共 9 页；再加一个首页（第0页）。  
> 所有页面都支持“演示模式”：用预置数据联动即可。

### 0）首页 Home
**目标**：3条产品线一眼看懂 + 1分钟案例入口  
**模块**：
- Product Suite Cards（3张大卡，每张：一句话价值 + 3个交付物标签 + 开始按钮）
- Demo Cases（快充车选型 / 新体系寿命预测 / 退役包评估）
- Recent Deliverables（最近报告卡片，点击进入“报告页”预览）

---

## 6. Turbo-Select（选型与边界评估）

### S1）工况输入（Scenario Builder）
**左栏输入**：
- 工况类型：车用快充 / 储能日循环 / 高温存储（下拉）
- 关键参数：温度范围、倍率、SOC窗口、脉冲/休眠
- 约束：成本、尺寸、供应链（可选）
**中栏展示**：
- 工况谱简图（可简化）
- 应力因子摘要卡（Top3：温度/倍率/SOC高位占比）
**右栏结论卡**：
- “已识别关键应力因子：xxx”
- CTA：`生成候选电芯列表`

### S2）推荐结果（Top-N + 解释）
**中栏展示**：
- Top 5 电芯榜（适配度分数 + 风险标签 + 推荐理由3条）
- 电芯对比（最多3个对比：能量密度/温域/倍率余量/成本）
**右栏结论卡**：
- 最佳候选：型号A（原因）
- 主要风险：xxx
- CTA：`查看 SOA 边界与策略`

### S3）SOA边界与策略（可执行交付）
**中栏展示**：
- SOA热力图/边界图（温度-倍率-SOC窗口的可视化即可）
- 避坑清单（触发条件→后果→建议动作）
**右栏结论卡**：
- 推荐运行窗口（参数表小卡：SOC上限、快充限流、低温预热阈值）
- CTA：`一键生成选型与策略报告`

---

## 7. Turbo-Dev（研发加速与工艺优化）

### D1）设计参数输入（New Cell Input）
**左栏输入**：
- 体系、容量、极片参数（可简化成“上传参数表/手动输入”两种）
- 目标：循环寿命目标、工况选择
**中栏展示**：
- 参数完整度进度条（让用户感觉“可计算”）
**右栏结论卡**：
- CTA：`极速预测（Online）`

### D2）极速预测（寿命区间 + 可解释性）
**中栏展示**：
- SOH/RUL 区间曲线（预置）
- 关键因子贡献（条形/列表：温度、倍率、SOC高位等）
**右栏结论卡**：
- 预计寿命：xx–yy（置信度）
- 主要风险：xxx
- CTA：`升级 VIP 深度检测（短测+仿真+PIML）`

### D3）VIP深度页（流程条 + 结果画廊）
**中栏展示**：
- Stepper：提交样品→短测试→仿真（微观）→PIML（宏观）→报告
- 结果画廊：温度场/浓度场/热点定位（示意图即可）
**右栏结论卡**：
- “深度结论摘要”（3条）
- CTA：`生成研发验证报告`

---

## 8. Turbo-Inspect（在役/退役评估与诊断）

### I1）数据导入（BMS / 历史数据）
**左栏输入**：
- 上传：BMS日志/循环记录/温度记录（拖拽区）
- 数据质量检查（缺失率、时间跨度）
**中栏展示**：
- 数据预览（抽样表/趋势小图）
**右栏结论卡**：
- CTA：`生成健康评估`

### I2）健康评估（可用性 + RUL）
**中栏展示**：
- 健康评分（A/B/C/D）+ 风险等级（低/中/高）
- RUL区间曲线（预置）
**右栏结论卡**：
- 判定：可用/谨慎使用/不可用（3态）
- CTA：`进入分流与诊断建议`

### I3）诊断与分流（证据链 + 根因树 + 去向建议）
**中栏展示**：
- Evidence Chain：现象→异常指标→推断原因（可折叠）
- 根因树（SEI增厚/锂沉积/气胀/极片问题…示意）
- 分流建议卡：继续使用 / 梯次利用 / 拆解重组 / 报废回收
**右栏结论卡**：
- 推荐去向（1个主建议 + 2个备选）
- CTA：`导出梯次评估/诊断报告`

## 10. 演示模式（Demo Mode）规则

### 10.1 演示数据联动（必须）
- 工况参数变化 → Top-N 推荐榜顺序变化
- 选择不同电芯 → SOA边界图变化
- 点击“导入10周短测（示例）” → 寿命区间变窄（动画/数值变化）
- 数据质量差 → 右侧结论提示“建议补充数据/升级VIP”

### 10.2 演示路径（60秒）
首页 Demo → S1 改温度 → S2 看Top-N → S3 看SOA策略 → 一键导出  
然后切到 D2（极速预测）→ D3（VIP流程墙）→ I2（健康评分）→ I3（分流建议）→ 导出报告。

## 11. 文案风格（按钮/提示语建议）

- CTA按钮统一动词开头：`生成` / `查看` / `导出` / `升级`

  1) 本页结论（1句话）
  2) 关键数字（1–2个）
  3) 下一步动作（1个主CTA）

示例：
- “本工况关键风险：高温+高SOC驻留”
- “预计寿命：1200–1600 cycles”

# TurboCell 极速芯研 Web 原型设计文档（浅色企业风 · 分层架构版）
> 目的：做一套“可路演演示”的平台原型。重点不是堆功能页，而是把 **Online 标准化产品**、**Offline/VIP 分级检测交付**、以及 **资产沉淀飞轮** 这三层清晰呈现出来。  
> 命名：三条产品线统一为 **Turbo-Select / Turbo-Dev / Turbo-Inspect**。

---

## 1. 产品本质与演示目标

### 1.1 我们到底在卖什么
- **工程决策确定性**：能不能用、怎么用才安全、寿命区间、主要风险、怎么改、为什么坏。
- **两种交付形态**：
  - **Online（标准化快评）**：快、便宜、可规模化
  - **Offline/VIP（分级检测交付）**：短测 + 诊断/拆解 + 仿真 + PIML + 证据链 → 参数级策略/工程报告

### 1.2 路演必须出现的“3个瞬间”
1) **改一个输入参数 → 结果立刻联动**（榜单/区间/风险等级变化）。  
2) **右侧结论卡给“可执行策略/分流建议”**（不是泛泛建议）。  
3) **一键导出交付物**（轻报告 / 工程交付包 / 证据包）。

---

## 2. 分层架构（强制写在文档最前面）

### L0 价值层：工程决策确定性
- 输出：Top-N 方案、SOA/SOA边界、寿命区间、风险Top3、分流建议、整改建议。

### L1 交付形态层：Online 产品 + Offline/VIP 项目
- Online：输入 → 快评 → 轻报告  
- Offline/VIP：选套餐 →（可选加测）→ 执行 → 证据链/仿真/PIML → 工程交付包

### L2 能力引擎层：数据/测试/仿真/PIML/证据链
- 数据资产：电芯库、工况库、案例库（拆解与证据包）、报告库
- 测试诊断：短周期验证、EIS/DRT、IC-DV、存储/浸润等（演示可写死）
- 仿真：机理定位（热点/沉积/副反应等）
- PIML：寿命区间、不确定性、关键因子贡献
- 证据链：现象 → 指标 → 推断 → 排除项 → 动作建议

### L3 工作流层：统一闭环
输入 → 分析 → 策略 → 报告 → 沉淀入库（Assets）

### L4 界面层：三大入口
- **Products（在线产品）**
- **Projects（VIP项目交付）**
- **Assets（资产库）**

---

## 3. 信息架构（IA）

### 3.1 顶部导航（Top Bar）
- 左：Logo + 当前项目下拉（Project Switcher）
- 中：全局搜索（项目 / 电芯 / 报告 / 案例）
- 右：演示模式 Demo Mode｜导出中心 Exports｜账号/权限

### 3.2 三大入口（一级导航）
1) **Products**：Turbo-Select Online / Turbo-Dev Online / Turbo-Inspect Online
2) **Projects**：新建VIP项目、测试计划、执行进度、交付物
3) **Assets**：电芯库、工况库、案例库（证据包）、报告库

> 说明：Turbo-Select/Dev/Inspect 是“产品线”；Online/Offline 是“交付形态”。UI 必须同时表达这两层。

---

## 4. 视觉风格（浅色企业风）
- 风格关键词：克制、可信、留白、结论优先
- 布局：桌面优先，12列栅格；卡片圆角小（6–10px），阴影弱
- 风险表达：不做大红大黄，使用“标签 + 小图标 + 浅底色”
- 组件一致性：相同交互在不同产品线一致（降低学习成本）

---

## 5. 页面通用布局与组件

### 5.1 通用三栏布局（推荐）
- 左栏：输入与参数（可折叠）
- 中栏：主结果（榜单/曲线/热力图/证据链）
- 右栏：结论卡 + CTA（导出/下一步）

### 5.2 必备组件库（用于“像平台”）
- Project Card（项目卡）
- KPI Tiles（关键指标卡：寿命区间/风险等级/适配度）
- Risk Tag（低/中/高）
- Stepper（流程：提交→执行→审核→交付）
- Report Builder（模板+章节勾选+导出）
- Evidence Chain（证据链：现象→指标→推断→排除项）
- Root Cause Tree（根因树）
- Add-ons Selector（可选加测：分类、推荐、依赖校验）

---

## 6. Products：三条产品线（Online）

> Online 只做“方向性结论 + 轻报告”，并且负责把用户引导到 Offline/VIP（Projects）。

### 6.1 Turbo-Select Online（选型与边界快评）
- 输入：工况谱/约束（温度、倍率、SOC窗口、尺寸、成本…）
- 输出：Top-N 候选电芯 + 初步边界提示（SOA粗版）+ 风险Top3
- CTA：导出《轻选型报告》 / “升级到VIP做可执行参数表”

### 6.2 Turbo-Dev Online（研发极速预测）
- 输入：新电池关键参数 + 目标工况
- 输出：寿命区间（粗）+ 关键因子贡献 + 主要风险
- CTA：导出《轻预测报告》 / “升级VIP做短测+仿真+PIML”

### 6.3 Turbo-Inspect Online（在役/退役初评）
- 输入：BMS/历史数据上传（CSV/表格）
- 输出：健康评分（A/B/C/D）+ RUL区间（粗）+ 分流建议（继续用/梯次/拆解/回收）
- CTA：导出《初评报告》 / “升级VIP做根因证据链”

---

## 7. Projects：Offline/VIP（分级套餐 + 可选加测 + 交付闭环）

### 7.1 Offline/VIP 的“分级套餐”（通用三档）
- **L1 Screening｜快速筛查**
  - 回答：有没有明显雷 / 是否值得继续
  - 交付：结论摘要 + 风险项清单 + 下一步建议
- **L2 Standard｜工程评估（推荐）**
  - 回答：边界/寿命区间收敛/主要风险
  - 交付：工程报告 + 参数级建议（策略/边界/分流）
- **L3 Deep-Dive｜根因与优化**
  - 回答：为什么 + 怎么改 + 证据链可复核
  - 交付：根因Top1–3（置信度）+ 证据包 + 优化方案

> 命名建议统一，不要写“套餐一/二/三”。核心是“确定性提升”。

### 7.2 可选加测（Add-ons）设计规则（必须产品化）
- 分类：电性能 / 阻抗诊断 / 材料表征 / 安全滥用 / 气体&水分 / 结构拆解
- 每项测试卡片必须包含：
  - 目的（回答什么问题）
  - 输出信号（图谱/指标）
  - 耗时等级（⚡⚡⚡）
  - 是否破坏性（Non-destructive / Destructive）
  - 适用场景标签（快充/高温存储/储能循环…）
- **推荐组合**：按场景与目标交付物自动勾选（演示可写死规则）
- **依赖校验（Guardrails）**：用户乱选时提示最低集合
  - 例：要输出“根因树”，必须包含某类诊断证据（如 EIS/DRT 或 IC-DV 等）
  - 例：要输出“SOA参数表”，必须包含短测验证模块

### 7.3 交付物反推（Deliverable-first）
Offline 不是“勾测试→出结果”，而是先选你想得到什么：
- 我想要 **SOA/策略参数表**
- 我想要 **根因Top3+证据链**
- 我想要 **寿命区间收敛到 ±X%**
系统自动推荐最小测试组合（演示用写死逻辑也可以）。

## 8. Assets：资产库（让“飞轮”在 UI 中可见）

### 8.1 电芯库（Cell Catalog）
- 来源：可直接导入《商用电芯信息列表.xlsx》
- 用途：Turbo-Select Online 的候选池；Projects 的对比与复用
- 页面：筛选/对比/详情（字段不全也可，演示重点是“可检索、可对比”）

### 8.2 工况库（Scenario Library）
- 用途：统一工况标签、复用工况谱、提高可比性
- 页面：工况模板（快充车/储能日循环/高温存储等）+ 一键引用到项目

### 8.3 案例库（Failure Cases / Evidence Packs）
- 每个案例包含：
  - 对外PDF（脱敏）
  - **结构化证据包 Evidence Pack**（现象→指标→机理→排除项→置信度）
  - 关键图谱/照片画廊
- 用途：
  - Turbo-Inspect Deep-Dive 的证据链与根因树调用
  - Turbo-Dev Deep-Dive 的“验证闭环”引用

### 8.4 报告库（Deliverables）
- 轻报告（Online）与工程交付包（Offline）分开标签
- 支持版本（v1/v2）与审核状态（演示可占位）

## 9. 报告体系（统一模板，但分层交付）

### 9.1 三类交付物定义
- **Light Report（Online）**：方向性结论 + 风险提示 + 推荐下一步
- **Engineering Package（Offline L2/L3）**：
  - 工程报告 PDF
  - 策略参数表（CSV/Excel）
  - 图谱附件（可选）
- **Evidence Pack（Offline L3）**：
  - 结构化证据（JSON/表格）
  - 关键图谱索引 + 截图

### 9.2 Report Builder（页面能力）
- 模板：Select/Dev/Inspect 各自模板
- 章节勾选：结论页、参数表、风险说明、证据链、附件
- 导出：PDF + 参数表 + 分享链接（只读，占位）

## 11. 演示模式（Demo Mode）联动规则（必须做）
- Select：温度/倍率变化 → Top-N排序变化 + 风险标签变化
- Dev：点击“导入短测样例” → 寿命区间变窄 + 置信度上升
- Inspect：数据质量差 → 提示“建议补数据/升级VIP”；数据完整 → 输出分流建议
- Offline：选择不同套餐 → 默认勾选的测试组合变化；勾选加测 → 右侧交付物预览增加对应图谱目录

## 12. 素材清单（实现原型所需最小集合）
- 电芯库：10–30条记录（可直接用你现有Excel导出）
- 工况模板：3套（快充车/储能日循环/高温存储）

- Offline 套餐配置：3档 + 每档默认包含项

- 报告样例：Online 轻报告 1份 + Offline 工程交付包 1份 + Evidence Pack 示例 1份
